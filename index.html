How to Use

1. Connect to your SPS5000X

Ethernet (recommended)
Run uv run bridge.py, select [1] Ethernet, enter the IP address shown on the device display, then click Bridge. The bridge connects via TCP to port 5025.
USB (Linux)
Run uv run bridge.py, select [2] USB. The SPS5000X presents as /dev/usbtmc0 on Linux. To grant access without sudo, add a udev rule:
echo 'SUBSYSTEM=="usbmisc", KERNEL=="usbtmc*", ATTRS{idVendor}=="f4ec", MODE="0666"' | sudo tee /etc/udev/rules.d/99-siglent-sps5000x.rules
sudo udevadm control --reload-rules && sudo udevadm trigger

2. Toolbar buttons

Play / Stop
Start or stop automatic polling. Sends 9 measurement queries per cycle to read voltage, current, and CV/CC mode for all three channels.
Record / Save
Record captures every reading with a timestamp. Save stops and downloads a CSV file with columns: Timestamp, CH1_V, CH1_I, CH2_V, CH2_I, CH3_V, CH3_I.

3. Controls

Expand Controls to set voltage and current limits for each channel. Use Output ON / OFF to toggle all channels simultaneously.

4. Charts

Two stacked charts show voltage and current for all channels (CH1 red, CH2 blue, CH3 green). Window sets how much history to show. Clear erases all data and resets statistics.

5. Demo mode

Click Demo to generate fake PSU data (CH1: ~5V/0.5A, CH2: ~12V/1.0A, CH3: ~3.3V/0.3A) and try out the interface without hardware.

6. Typical workflow

Connect → set voltage/current limits → Output ON → Record → Play → monitor live readings → Stop → Save to download CSV.

Running locally

Don't open index.html directly — ES modules need a web server. Run uv run serve.py and open http://localhost:8000.

SCPI Command Reference

Measurement Queries (per channel)

CommandReturnsDescription
MEASure:VOLTage? CH12.991442Actual output voltage
MEASure:CURRent? CH11.999407Actual output current
MEASure:RUN:MODE? CH1CVOperating mode (CV or CC)
*IDN?ID stringDevice identification

Set Commands

CommandDescription
:SOURce:VOLTage:SET CH1,5.000Set CH1 voltage setpoint
:SOURce:CURRent:SET CH1,2.000Set CH1 current limit
OUTPut ONEnable all channel outputs
OUTPut OFFDisable all channel outputs

Communication

Ethernet TCP on port 5025. USB USBTMC on Linux (/dev/usbtmc0). Commands terminated with \n. Responses terminated with \n. Replace CH1 with CH2 or CH3 for other channels.
